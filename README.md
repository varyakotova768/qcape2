# Дашборд загруженности команды КуСкейп

Дашборд для визуализации загруженности сотрудников с интеграцией через API Битрикс24. Позволяет отслеживать распределение задач, выявлять перегруженных и свободных сотрудников, анализировать статусы задач в выбранном временном периоде.

---

## 📋 Функциональные возможности

- **Просмотр загруженности сотрудников** — для каждого сотрудника отображается список активных задач и его нагрузка относительно рабочего времени за выбранный день / период.
- **Фильтрация по периоду** — день, период, готовый выбор: сегодня, текущая неделя, теукщий месяц, текущий год.
- **Фильтрация по статусу загрузки** — все / перегружены / норма / свободны.
- **Вкладка «Задачи»** — список задач с фильтрацией по статусу и поиском.
- **Режим «Без времени»** — показывает задачи, у которых отсутствуют временные метки (дата начала/окончания или плановое/затраченное время) из-за чего нельзя посчитать их влияние на загруженность.
- **Статистические карточки** — общее количество сотрудников, перегруженных, свободных, задач и процент выполненных.
- **Диаграмма распределения** — визуальное отображение соотношения перегруженных / с нормальной загрузкой / свободных сотрудников.
- **Интеграция с Битрикс24** — данные загружаются через REST API, есть кнопки для перехода в карточку сотрудника или задачи.

## 🚀 Быстрый старт

### 1. Установка Node.js

Перед началом работы убедитесь, что у вас установлена **Node.js версии 20.19.0 или выше**.

```bash
node -v
```

Если Node.js не установлена, скачайте её с официального сайта.

### 2. Клонирование проекта

```bash
git clone https://github.com/varyakotova768/bitrix24-dashboard-qscape.git
cd bitrix24-dashboard-qscape
```

### 3. Установка зависимостей

```bash
npm install
```

### 4. Настройка переменных окружения

Для работы с Битрикс24 необходим вебхук.

1. В корне проекта есть файл-шаблон `.env.example`:
2. Создайте свой файл `.env`:

```bash
cp .env.example .env
```

3. Откройте `.env` и укажите ваш вебхук:

```
VITE_BITRIX_WEBHOOK=https://ваш-битрикс24.bitrix24.ru/rest/1/ваш_секретный_ключ/
```

**Где взять вебхук?**
- Зайдите в Битрикс24
- Раздел **«Разработчикам» → «Вебхуки»**
- Создайте новый вебхук или используйте существующий
- Для работы вебхука необходимо настроить права: Задачи (task), Задачи (tasks), Пользователи (user)
- Не допускайте попадание вашего вебхука в общий доступ

### 5. Запуск проекта

**Режим разработки (с автоматической перезагрузкой):**

```bash
npm run dev
```

После запуска откройте в браузере: http://localhost:5173/

**Сборка для продакшена:**

```bash
npm run build
```

**Просмотр собранной версии:**

```bash
npm run preview
```

**Деплой на GitHub Pages:**

```bash
npm run deploy
```

## ⚙️ Как это работает: логика расчёта загруженности

### Источники данных

- **Сотрудники** — загружаются через метод `user.get`.
- **Задачи** — загружаются через `tasks.task.list` с полным набором полей (`ID`, `TITLE`, `STATUS`, `RESPONSIBLE_ID`, `TIME_ESTIMATE`, даты и др.).
- **Затраченное время** — для каждой задачи запрашивается через `task.elapseditem.getlist`.

### Расчёт загрузки

1. Для каждой задачи определяется **интервал выполнения**:
   - Начало: `plannedStart` или `createdDate`.
   - Конец: для завершённых — `closedDate`, иначе `plannedEnd` или `deadline`.

2. Задача **попадает в период**, если её интервал пересекается с выбранным диапазоном дат.

3. Учитывается только **время** задачи:
   - Для завершённых — `timeSpent` (фактическое), либо `timeEstimate` (плановое).
   - Для активных — `timeEstimate`.

4. Если интервал задачи выходит за пределы выбранного периода, время **пропорционально пересчитывается** по числу рабочих дней пересечения.

5. **Загрузка сотрудника** = суммарное время всех его задач в периоде (в часах).

6. **Статус загрузки**:
Статус загрузки определяется путём сравнения запланированных часов с количеством рабочих часов в выбранном периоде:
- если загрузка менее 93% — статус «свободен» (есть свободные рабочее время);
- если загрузка в диапазоне 93%–107% — статус «норма»;
- если загрузка более 107% — статус «перегружен» (требуется делегировать задачи).

Это позволяет более точно выявлять сотрудников, чья нагрузка отклоняется от плановой более чем на 7%.

### Режим «Без времени»

Показывает задачи, у которых отсутствуют:
- дата начала или окончания,
- плановое или затраченное время.

Это помогает выявить задачи с неполными данными, требующие внимания. У таких задач недостаточно данных для оценки их влияния на загруженность.

## ❗ Частые проблемы и их решение

### 1. Ошибка «VITE_BITRIX_WEBHOOK не задан»

**Решение:** Проверьте, что файл `.env` существует и в нём правильно указан вебхук.

### 2. Ошибка при запуске «node: command not found»

**Решение:** Установите Node.js с официального сайта.

### 3. Ошибка при установке зависимостей

**Решение:**

```bash
rm -rf node_modules
rm package-lock.json
npm install
```

### 4. Не показываются сотрудники из Битрикс24

**Решение:**
- Проверьте правильность вебхука в `.env`
- Откройте консоль браузера (F12) и посмотрите ошибки
- Убедитесь, что вебхук имеет доступ к методам `user.get` и `tasks.task.list`

## 👩‍💻 Для разработчиков

### Как добавить новый компонент

1. Создайте файл в папке `src/components/`
2. Импортируйте и используйте в нужном месте

### Как изменить стили

- Глобальные стили — в `src/main.css`
- Стили компонента — внутри `<style scoped>` в самом компоненте

### Как изменить логику расчёта загрузки

Все утилиты для работы с задачами и датами находятся в `src/services/taskHelpers.js`:
- `getTaskInterval` — определение интервала задачи
- `getTaskTime` — получение времени задачи
- `isTaskInPeriod` — проверка попадания в период
- `getWorkingDaysCount` — подсчёт рабочих дней
